## 스냅샷
- 마치 폴라로이드 카메라로 풍경을 기록하듯이 **파일의 상태를 기록**하는 시스템
- **한 번 기록하면 변경 불가능**

## 3가지 영역
- **작업 트리(Working Tree)**: 변경 이력이 있는 파일이 위치한 영역
- **스테이징 영역(Staging Area)**: 작업 트리에 위치한 파일이 로컬 저장소로 넘어가기 전에 위치하는 가상의 영역
- **로컬 저장소(Local Repository)**: 파일이 저장되는 영역. 해당 영역에 위치된 파일은 수정 불가능

## 파일의 3가지 상태
- **수정**: 작업 트리에 위치 되어 있는 파일의 상태
- **스테이징**: 수정 상태의 파일을 저장하기 전에 임시 영역에 올려놓은 상태
- **커밋**: 파일이 저장소에 저장된 상태

# 깃 명령어

## 변경된 파일을 스테이징 영역으로 이동하는 명령어
- **`git add "파일명"`** : 변경된 단일 파일을 스테이징 영역으로 이동
- **`git add .`** : 변경된 전체 파일을 스테이징 영역으로 이동
## 스테이징 영역에 있는 파일을 기록(커밋)하는 명령어
- **`git commit`**
- **`git commit -m "커밋메시지"`** : 커밋 메시지 내용 작성과 커밋을 한 번에 가능
## 현재 파일 상태를 출력하는 명령어
- **`git status`**

# .gitignore

## 어떤 역할을 하는 파일인가?
- git을 사용하여 프로젝트를 진행할 때, 외부로 노출되면 안되는 데이터(파일)를 관리하는 파일
## .gitignore 사용 시 주의사항은 무엇인가?
- 한번 저장(commit)이 된 파일은 .gitignore 파일에서 관리 불가능
## gitignore.io는 어떤 서비스인가?
- 프로젝트 초기 설정 시 git을 통한 관리에 포함하면 안되는 파일을 설정해주는 서비스

# 깃허브
## 원격 저장소 생성, 연결, 동기화
1. 원격 저장소 생성
2. 지역 저장소 -> 원격 저장소 연결
3. 지역 저장소 -> 원격 저장소 push
4. 원격 저장소 파일 생성
5. 지역 저장소 <- 원격 저장소 pull

## git push origin main, origin은 무엇인가?
- `git push <원격저장소명> <브랜치명>`
- `git remote add <원격저장소명> <원격저장소주소>`
- `origin` 이라는 원격저장소명은 관습적인 이름
  - 가급적이면 `origin` 이라는 이름 사용하기

## 원격 저장소란 무엇인가?
로컬 저장소와 물리적으로 떨어져 있는 저장소
## 지역 저장소와 원격 저장소의 구분
- 지역저장소: 개인 작업 공간
  - 개인 작업용 컴퓨터(PC)에서 작업
- 원격저장소: 공동 작업 공간
  - GitHub, GitLab 등의 원격 서버에서 작업

## 지역 저장소와 원격 저장소의 동기화란 무엇인가?
지역저장소와 원격저장소 사이에 연결루트를 만들어서 서로의 변경사항을 적용하는 방법
> 동기화 사용 방법: `push`, `pull`

# git reset

## git reset은 어떤 명령어인가요?
원하는 커밋 시점으로 되돌리는 명령어

## git reset의 3가지 옵션은 무엇이고, 각각의 의미는 무엇인가요?
- **`--soft`** : 커밋된 항목을 스테이징 상태(스테이징 영역)로 되돌린다.
- **`--mixed`** : 커밋된 항목을 수정(워킹트리) 상태로 되돌린다.
- **`--hard`** : 커밋된 항목에 대한 내용 자체를 삭제한다(파일까지).

# Git 정리
## `git`
코드의 변경사항을 관리해주는 개발 도구.

## `git`의 3가지 영역
- **워킹 트리(Working Tree)** : 변경된 파일이 위치해 있는 영역.
- **스테이징 영역(Staging Area)** : 변경된 파일이 저장소로 넘어가기 전에 임시적으로 위치해 있는 영역.
- **로컬 저장소(Repository)** : 변경된 파일이 저장되는 영역.

## `git`의 3가지 상태
- **수정** : 워킹 트리에 있는 상태. 파일과 같은 데이터가 변경되어 있는 상태.
- **스테이징** : 변경된 파일이 저장소로 넘어가기 전 스테이징 영역에 임시적으로 머무르는 상태.
- **커밋** : 변경 상태의 파일이 저장소에 저장된 상태.

## 원격저장소(Remote Repository)
- 로컬과 물리적으로 떨어져 있는 저장 공간.
- `Github`, `GitLab` 등이 있다.

## 로컬 저장소와 원격 저장소의 동기화
로컬 저장소와 원격 저장소 사이에 연결 루트를 구축하여 각각의 변경사항을 적용하는 과정.

- **`git push <원격저장소명> <브랜치명>`** : 로컬 저장소의 변경사항을 원격 저장소에 적용하는 명령어.

- **`git pull <원격저장소명> <브랜치명>`** : 원격 저장소의 변경사항을 로컬 저장소에 적용(당겨오는)하는 명령어.

## git clone
다른 유저의 원격 저장소 정보를 **복제**하는 방법이다.

### 1. 특징
해당 원격 저장소의 git 관리 히스토리를 그대로 가져온다.

### 2. 명령어 구조
`git clone <원격저장소주소>`

## .gitignore
git으로 관리하는 것을 원치 않는 파일들을 설정하는 파일

### gitignore.io
- 편리하게 .gitignore에 추가할 파일들을 정리해주는 사이트

### 사용 시 주의 사항
한 번 저장된(commit) 파일에 대해서는 `.gitignore` 에서 관리가 불가능하다.

- 한 번 기록하면 변경 불가능(사진과 유사).
